### Task 2: BFS та DFS

- **BFS** - цей алгоритм використовуєтсья для пошуку шляху з мінімальною кількістю кроків
- **DFS** - цей алгоритм використовується для перевірки існування шляху між двома вершинами.

```bash
python task2.py PrivatBank Binance
```

**Результат для PrivatBank -> PayPal:**

| Алгоритм | Результат |
|----------|-----------|
| BFS | PrivatBank -> Monobank -> Revolut -> PayPal (3 кроки) |
| DFS | Шлях існує |

### Task 3: Алгоритм Дейкстри

Пошук найдешевшого маршруту (мінімальна сумарна комісія).

```bash
python task3.py PrivatBank PayPal
```

**Порівняння для PrivatBank -> PayPal:**

| Метод | Шлях | Вартість |
|-------|------|----------|
| BFS (найменше кроків) | PrivatBank -> Monobank -> Revolut -> PayPal | 9$ |
| Дейкстра (найдешевше) | PrivatBank -> Monobank -> Revolut -> PayPal | 9$ |

## Висновки

1. **BFS** знаходить шлях з мінімальною кількістю ребер (кроків), не враховуючи ваги.

2. **DFS** швидко перевіряє існування шляху, але не гарантує оптимальність.

3. **Дейкстра** знаходить шлях з мінімальною сумарною вагою (комісією).

4. У даному графі BFS та Дейкстра можуть дати однаковий результат, якщо найкоротший шлях за кількістю ребер також є найдешевшим.

5. Ізольована вершина (I_AM_ALONE) показує випадок, коли шлях не існує.
